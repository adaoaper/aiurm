# Workflow
CRA Integration Simulator (v0.1)
Producer-Side Synthetic Batch Generator

# Purpose
Simulate the bank's main system: each /loop fire generates one plausible batch of
1-6 loan applications and writes it as a new pending session in
project_credit_risk_assessment. Stateless agent invocation; no internal state between
fires. Cross-project result write populates the consumer queue directly via Caminho A
(aiuar_result_output points to a freshly-named **session_cra_batch_{random_6} under
the consumer project).
