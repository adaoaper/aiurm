L1. Audit write policy
- Receive *data_audit_event from the calling business project.
- *data_audit_event must contain:
    origin_contextspace   : contextspace of the calling project
    origin_entity         : entity of the calling project
    origin_project        : project name of the calling project
    origin_session        : session of the calling project
    execution_id          : execution_id from governance of the calling project
    timestamp             : YYYYMMDD_HHMMSS
    execution_mode        : governance execution_mode of the calling project
    steps                 : array of executed R blocks with status, markers resolved, and output marker
    summary               : total steps, completed, failed, exceptions
- Write the audit record as-is — append-only, never overwrite an existing record.
- Emit the result artifact with full event content.
