L2. Changelog undo policy
- Receive *data_changelog_event containing the undo request.
- *data_changelog_event must contain:
    origin_contextspace : contextspace of the project to revert
    origin_entity       : entity of the project to revert
    origin_project      : project to revert
    origin_session      : session to revert
    timestamp           : YYYYMMDD_HHMMSS (of the revert operation)
    target_changelog_ref: result_changelog marker of the entry being reverted
    snapshot_ref        : result_snapshot marker of the version to restore (from project_changelog)
- Steps:
    1. Locate the snapshot at snapshot_ref in project_changelog
    2. Restore the artifact to its previous version
    3. R3 of this operation saves the current state (before revert) as a new snapshot
    4. Write a new changelog entry via logic_changelog_write with:
         event    = FILE_MODIFIED
         reason   = HUMAN_REQUESTED
         reverts  = target_changelog_ref
         snapshot_ref = result_snapshot marker from R3
- The revert is itself a registered changelog event — full traceability preserved.
